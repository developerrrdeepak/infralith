### Steps to Fix "503 Service Unavailable":

1.  **Correct App Name**:
    Ensure your GitHub Workflow matches the App Service name. I have updated `.github/workflows/main_infralith.yml` to specifically target `infralith-epd3e7akbyf0a2ce`.

2.  **Startup Command**:
    The most common cause of 503 errors with Next.js on Azure is an incorrect startup command.
    *   Go to the [Azure Portal](https://portal.azure.com).
    *   Navigate to your App Service: `infralith-epd3e7akbyf0a2ce`.
    *   Go to **Settings > Configuration > General settings**.
    *   Set the **Startup Command** to: `node server.js`
    *   Click **Save**.

3.  **Environment Variables**:
    Ensure the following "Environment variables" (under Configuration) are set in the Azure Portal:
    *   `PORT`: `8080` (Azure usually maps this automatically, but explicit is better).
    *   `NEXTAUTH_URL`: `https://infralith-epd3e7akbyf0a2ce.centralindia-01.azurewebsites.net`
    *   `NEXTAUTH_SECRET`: (Get this from your `.env.local` or generate a new one).
    *   `AZURE_OPENAI_ENDPOINT`, `AZURE_OPENAI_KEY`, etc. (All keys from your `.env.local`).

4.  **Redeploy**:
    Push a change to the `main` branch. The `Build and deploy Node.js app to Azure Web App - infralith-epd3e7akbyf0a2ce` workflow will trigger.
